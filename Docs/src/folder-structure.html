Folder Structure

Described below the folder structure of template and what is contains.

Understanding the src folder

The src folder contains all the assets, pages, constants, helpers, redux-store etc files.

  • src/assets/ folder containes all fonts, images & scss files used in template.
  • src/components/ folder containes templates common components used on several pages.
  • src/constants folder contains the all constants of the template i.e. layout constants. you can add more constant file here. The constants folder added to the template for better performance, let's say if we will require to update value of some variable which is used in many functionalities, then we do not need to update it's value everywhere, we will just update the variable's value locatend in this folder, and use these constant in relatable functionality.
  • src/data contains dummy data files used in templates.
  • src/helpers/ folder contains functional files related to api integration, notifications etc. i.e. firebase, api core, etc.
  • src/hooks contains custom hooks used in the template. e.g. useProfile hook etc.
  • src/routes contains route configutation & all routes for the template.
  • src/utils contains the utility functions used in several modules.
  • src/pages contains template's pages, you can create/add/remove you pages here.
  • src/redux contains template's global store. check Global State page for more details.
